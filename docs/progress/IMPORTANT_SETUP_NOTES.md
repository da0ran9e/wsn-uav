## Python Version Requirement

### ❌ Problem: Python 3.14 Incompatibility

If you see this error when running `./ns3 configure`:

```
ValueError: action 'store_true' is not valid for positional arguments
  File "/Users/mophan/Github/ns-3-dev-git-ns-3.46/./ns3", line 129, in parse_args
    parser_help.add_argument(
        "help", help="Print a summary of available commands", action="store_true"
```

**Root Cause:** NS-3's `ns3` wrapper script has a Python 3.14 compatibility bug.

**Current Status:**
- ❌ **Python 3.14.3** → Error (do NOT use)
- ✅ **Python 3.13.x** → Works fine
- ✅ **Python 3.9.6** → Works fine

### ✅ Solution: Use Python 3.13

```bash
# Check current version
python3 --version

# If you have 3.14, downgrade to 3.13
brew install python@3.13
brew unlink python@3.14
brew link python@3.13

# Verify
python3 --version  # Should show 3.13.x
```

After switching Python version, these commands will work:
```bash
./ns3 configure --enable-examples --enable-modules=wsn-uav
./ns3 build
./ns3 run "scenario-1-single-uav --gridSize=10 --seed=1 --runId=1"
```

## Working Directory Boundaries

### 🔴 DO NOT MODIFY

❌ **Never modify files outside `src/wsn-uav/` directory:**
- `/ns3` wrapper script (Python version issue is in NS-3 core, not our code)
- Any other NS-3 framework files
- Core CMake configuration files outside wsn-uav

**Why?**
- These are part of the NS-3 framework, not the wsn-uav project
- Changes can be lost when updating NS-3 or cause conflicts
- If bugs exist in NS-3 framework, they should be reported to NS-3 maintainers, not patched locally

### ✅ DO MODIFY

✅ **Only work within `src/wsn-uav/` directory:**
```
src/wsn-uav/
├── CMakeLists.txt              ✅ Modify (our build config)
├── examples/                   ✅ Modify (our examples)
├── models/                     ✅ Modify (our models)
├── helper/                     ✅ Modify (our helpers)
├── docs/progress/              ✅ Modify (our documentation)
└── results/                    ✅ Modify (output generated by our code)
```

## If You Get Other Errors

1. **CMake configuration fails** → Run `./ns3 clean` first
2. **Build fails** → Make sure Python 3.13.x is active
3. **Simulation not found** → Verify `build_lib_example()` is used in `examples/CMakeLists.txt`
4. **Invalid parameter error** → Check parameter types (e.g., runId must be numeric)
